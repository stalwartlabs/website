# CLAUDE.md

Guidance for Claude Code working in this repo.

## What this is

The Stalwart Labs documentation site (https://stalw.art/docs). It is a [Docusaurus 3](https://docusaurus.io/) project; documentation is written in Markdown under `docs/` and served by Docusaurus. The deployed site is the documentation for the Stalwart mail and collaboration server (JMAP, IMAP, POP3, SMTP, CalDAV, CardDAV, WebDAV; written in Rust).

The repo is published at `stalwartlabs/website`. The README still mentions Docusaurus 2 and `yarn`, but the project is on Docusaurus 3.7 and the team uses npm — see `package-lock.json`.

## Commands

```
npm install         # install deps (Node >= 20, see .nvmrc)
npm start           # local dev server with hot reload
npm run build       # build static site to build/
npm run serve       # serve the built site locally
npm run clear       # clear Docusaurus cache
npm run deploy      # GitHub Pages deploy via docusaurus deploy
```

`build.js` is a separate, hand-written EJS compiler that turns templates in `html/*.ejs` into `static/<name>/index.html` (e.g. landing pages and shared nav HTML included by Docusaurus pages). It is not invoked by `npm run build`. Run it directly when editing EJS:

```
node ./build.js                                  # one-off
chokidar "**/*.ejs" -c "node ./build.js"        # watch mode
```

`html/include-*.ejs` files are partials and are skipped by `build.js`.

## Layout

- `docs/` — current documentation (the v0.16 line; mapped to the empty path in `docusaurus.config.js`).
- `versioned_docs/version-0.15/` and `versioned_sidebars/version-0.15-sidebars.json` — frozen snapshot for v0.15. Listed in `versions.json`. **Do not edit `versioned_docs/` for normal doc changes** — they are immutable snapshots of past releases. Only touch them for genuine corrections to historical docs.
- `blog/` — MDX blog posts; authors in `blog/authors.yml`.
- `src/pages/` — custom React/MDX pages. `docs/index.js` (or root index) redirects `/` to `/docs/install/get-started`.
- `src/css/custom.css` — Stalwart branding (pink `#DB2D54`, Inter, JetBrains Mono).
- `static/` — static assets served as-is, plus the HTML produced by `build.js` from `html/`. Also contains the agent-discoverability files: `robots.txt` (allow-all, advertises `/sitemap.xml`) and `llms.txt` (curated index of the docs, following the llmstxt.org format). Keep `llms.txt` in sync when adding or renaming top-level doc sections.
- `html/` — EJS templates feeding `build.js`; not consumed by Docusaurus directly.
- `build/` — Docusaurus build output (gitignored).
- `sidebars.js` — single autogenerated sidebar (`{type: 'autogenerated', dirName: '.'}`); structure comes from the filesystem and `_category_.json` files.
- `docusaurus.config.js` — Docusaurus config. `onBrokenLinks: 'throw'`, so broken internal links fail the build.

## Versioning

`versions.json` lists released-and-frozen versions (currently `["0.15"]`). The "current" version (the working set under `docs/`) targets the next release line — at the time of writing, v0.16, with v1.0.0 expected in H1 2026.

To cut a new version, use `npm run docusaurus -- docs:version <X.Y>`; this snapshots `docs/` into `versioned_docs/version-<X.Y>/` and `sidebars.js` into `versioned_sidebars/version-<X.Y>-sidebars.json`, and adds the version to `versions.json`. Don't do this without asking the user.

## Doc conventions

- Markdown frontmatter is YAML; ordering is via `sidebar_position: N`.
- Categories use `_category_.json` (label, position, optional `link.type: generated-index`).
- Admonitions: `:::tip`, `:::info`, `:::note`, `:::warning`, `:::danger` — sometimes with a title (e.g. `:::tip Enterprise feature`).
- Internal doc links use absolute Docusaurus paths: `[Foo](/docs/path/to/page)`. Anchors use `#kebab-case-heading`.
- TOML is the standard configuration language shown in code blocks (Stalwart configs are TOML); TOML highlighting is enabled.
- Mermaid diagrams are enabled via `@docusaurus/theme-mermaid`.

## Search and analytics

- Search is **Algolia DocSearch** via the `@docsearch/docusaurus-adapter` plugin (see `docusaurus.config.js`: `appId: CMWSXETHDL`, `indexName: 'stalw'`, contextual search on). The repo also has past Meilisearch integration in commit history.
- Analytics: `@docusaurus/plugin-google-gtag` with tracking ID `G-NTKTV3G55G`.
- Sitemap plugin enabled, excluding `/tags/**`.

## House style for documentation prose

Based on a cleanup pass done in this branch:

- **No em dashes** (`—`). Replace with comma, semicolon, colon, parentheses, or split into two sentences depending on context.
- **Avoid marketing/AI-speak**: words like *robust*, *seamless(ly)*, *leverage*, *empower(s)*, *unlock(s)* (when not the HTTP method), *streamline(s)*, *harness*, *delve*, *cutting-edge*, *state-of-the-art*, *lightning-fast*, *unparalleled*, *world-class*, *enterprise-grade*, *comprehensive* (when puffery), *powerful* (when puffery), *effortlessly*. Prefer concrete technical wording: *use*, *allows*, *enables*, *simplifies*, *reliable*, *fast*, *flexible*, etc. Watch for these especially in overview/intro paragraphs, which were largely AI-generated.
- Don't add filler like "By leveraging X, administrators can..." — describe the mechanism directly.
- Keep RFC and spec references precise; verify when in doubt.

## Things to verify before recommending a setting or path

- The Stalwart server source itself is **not** in this repo. If you are tempted to assert behavior of a setting (default value, valid keys, semantics), verify against the docs that already exist or ask the user — don't guess. The docs are the spec for users; don't introduce inaccuracies.
- Check `onBrokenLinks: 'throw'` will catch typos in `/docs/...` links at build time. Run `npm run build` to validate before claiming a doc change is complete.
