# Overview

The management of Stalwart can be approached through various methods, each suited to different user preferences and requirements:

The primary method is the [web-based admin tool](/docs/management/webadmin/overview), which is the recommended approach for configuring and managing Stalwart. This tool provides a full interface for overseeing the mail server and is accessible to non-technical users.

For users who prefer a hands-on method or require automation and scripting, Stalwart offers a [command-line interface](/docs/management/cli/overview) (CLI) tool. The CLI allows significant control over the server's operation and configuration, but it does not cover the entire range of functionality available through the web-based admin tool.

Additionally, catering to developers and those integrating Stalwart into larger systems or applications, there is a [REST API](/docs/api/management/overview). This API facilitates remote management and configuration of Stalwart through programmable interfaces, allowing for automation and integration within custom applications or management systems. This is particularly useful for those who need to embed mail server management directly into their software, offering a flexible way to control Stalwart programmatically.
